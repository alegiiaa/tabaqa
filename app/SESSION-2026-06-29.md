## How it works now (quick reference)
- **Data is always live.** The report re-fetches the engine so numbers match: demo → `scoreConnection`,
  upload → `scoreStatement` + `affordability({verified_income,…})`. Affordability feeds the prose
  financing line; if it fails the clause is omitted.
- **Self-contained links** (`reportlink.ts`): `encodeStatement`/`decodeStatement` (the `?d=` payload),
  `encodeFacts`/`decodeFacts` (the compact `?v=` verify token), `reportHref`, `shortHash`.
- **Verify modes:** `?c=` re-fetch (demo), `?d=` re-score (uploaded link), `?v=` show issued facts
  (the QR — always small/scannable).
- **One-page trick:** no variable-length table ⇒ deterministic height; `.rpt-body` is a flex column with
  `.rpt-bottom{margin-top:auto}` pinning the footer, watermark fills the middle whitespace.
- 4 demo connections: `con_8842` (Fahd, 82/low), `con_gig_driver`, `con_sme_owner`, `con_thin_file`.

## Run / deploy
- **Local:** API on `:8000` (FastAPI, from `app/`), web `cd app/web && npm run dev` on `:5173`
  (Vite proxies `/v1` → `:8000`). Report preview: `localhost:5173/report?c=con_8842` → ⌘P should read
  "1 sheet of paper".
- **Deploy web (from repo root `tabaqa/`):**
  `vercel deploy --prod --yes -b VITE_API_BASE=https://tabaqa-api.vercel.app`
  ⚠️ Do **not** run `vercel link`/`vercel pull` — it clobbers `app/web/.env.local`.
- Build check: `cd app/web && npm run build` (tsc + vite).
